## Build / test

- `make` = optimized release (`-O3 -ffast-math -fno-finite-math-only`, always rebuilds). `make sanitize` = ASan (heap/UAF). `make profile` = `-DAL_PROFILE` per-stage timing. Knobs: `CNAME=gcc-15`, `OMP=0`, `ZSTD=1`. Deps: zlib; libomp for OpenMP with Apple Clang.
- **`-ffast-math -fno-finite-math-only` is REQUIRED** (niimath's AFASTMATH). Plain `-ffast-math` implies `-ffinite-math-only` → the compiler assumes no NaN/Inf → breaks non-finite filtering. So finiteness uses magnitude guards (`al_finitef`: `-FLT_MAX ≤ v ≤ FLT_MAX`), NOT `isfinite()`.
- **Apple Silicon: ASan can't detect leaks and DEADLOCKS with libomp** (see `~/.claude/CLAUDE.md`). `make sanitize` builds serial (no libomp) on purpose. Leak-test the RELEASE binary: `MallocNanoZone=0 leaks --atExit -- ./allineate <mov> <fix> out.nii.gz`.
- **`make test`** is the correctness gate: `test/test_regression.py` (synthetic geometry/preprocessing/affine fixtures + fast-engine capture, cross-modal recovery, `-cmass`/`-nocmass` behavior, `-master` equivalence, and direct C-API boundaries) + `test/test_coreg_fast.py` (generated-phantom `-cost fastcr` capture recovery) + focused benchmark/NMS/NEWUOA helper tests. The C-API harnesses override `AL_SIZE_MAX=UINT32_MAX` to exercise wasm32 float-buffer limits without allocating a giant image. Exits nonzero on any failure; run before a release and extend it when adding a feature. Results must stay bit-identical at `-p 1` vs `-p N`.
- **Self-contained**: a clean clone needs the eight sources (`main.c allineate.c nifti_io.c powell_newuoa.c miniCoreFLT.c coreg_fast.c qwarp.c reface.c` + headers, including `core32.h`, `al_thread_local.h`, `al_size_guard.h`, `coreg_fast_nms.h`, `qwarp.h`, and `reface.h`), tests, `Makefile`, and this file. The gate generates its own fixtures.
- **`simbrain.py` (lives in niimath, not bundled)** — a stdlib-only synthetic T1w head phantom generator (`niimath/js/src/wasi/simbrain.py`) with a KNOWN rigid pose (`--translate`/`--rotate`) and optional partial FOV (`--fov-z`); a handy no-template `-cost fasthel`/`hel` sanity check. **Do not re-copy it here** — `make test` stays real-template-based via `test/coreg_synth.py`, so a local copy is just a hand-synced duplicate. Run it from the niimath checkout when you want a synthetic pair.
- `benchmark/benchmark.py` runs the `allineate` and `fast` engines (and AFNI `3dAllineate` with `--afni`) at 1 and N threads, scoring quality with a cross-modal-valid **Hellinger-affinity** metric. Descriptive (speed/quality only), not a correctness gate. See `benchmark/README.md`.

## Fast engine (`-cost fast` / `fastx` / `fasthel` / `fastcr`) — `coreg_fast.c/.h`

**This is now the default engine and strategy** — a bare `allineate <mov> <fix> out`, `-cost fast`, and `-cost fastx` all select the adaptive fast strategy. Select HEL-only fast registration with `-cost fasthel`, CR-only with `-cost fastcr`, or the ordinary AFNI-style engine with `-cost hel`/`nmi`/`lpc`/`lpa`/`ls`. Independent SPM/FLIRT-**inspired** clean-room implementation (never derived from `niimath/src/GPL/`). Hierarchical 8→4→2 mm pyramid. **Costs (`CF_COST_*`):** default `HEL_CR`; `HEL` (clean-room Hellinger-affinity), `CR` (correlation ratio), and diagnostic `LS` (Pearson). On a whole-head base, mixed capture independently fits the rigid 8 mm stage from supplied-affine/COM frames with both HEL and CR, selects the four candidates by HEL dependence×overlap, then runs the 4/2 mm stages once with HEL. A hard-zeroed base instead activates the full-depth multi-start described below. **Caveat:** HEL is imprecise on smooth synthetic phantoms (near-flat MI landscape), so the synthetic capture suites use CR; HEL is for real sharp-edged images. Returns world-mm `fixed_to_moving`; does not mutate inputs; host calls remain serial. `make profile` prints per-level timing (zero release overhead).

**Do-not-regress gotchas (each was a real bug or hard-won result):**
- **Thread-count-independent reduction**: samples split into a FIXED `CF_CR_NCHUNK` (64) chunks combined in fixed order → `-p 1`==`-p N` bit-identical. Do NOT use per-thread-count blocks (NEWUOA amplifies FP regrouping past the 0.05 mm floor).
- **Whole-head default/`fastx` coarse candidates parallelize at the optimizer level.** Normal auto mode has four independent jobs (HEL×affine, HEL×COM, CR×affine, CR×COM); `-com`/`-nocmass` reduce this to two. Every job owns a shallow `cf_ctx` plus private histogram scratch and NEWUOA TLS. Its inner cost loops are forced serial so the outer OpenMP loop owns up to four workers and cannot create nested teams/oversubscription. `g_cf` is TLS under OpenMP; aggregation and HEL re-scoring remain in fixed candidate order, preserving `-p1==-pN` bit identity. Each worker captures and restores its prior `mfac`/`afac`, so an internal job does not leak caller sampling configuration into a persistent OpenMP worker. The hard-zero multi-start instead uses the ordinary inner-cost parallelism for its serial full-depth strategies. `fasthel`/`fastcr` remain genuinely single-cost: each runs only its rigid and scale-bracketed trajectories, never the other cost's strategy.
- **Hierarchical pyramid**: each coarser level is built from the next-finer one (quadrature-added blur), not re-blurred from full-res.
- **Level dims cover the source FOV** (`ceil(n·vox/sep)`, not round). Bound the world-FOV product before the `int` cast (a finite-but-extreme sform/pixdim would be UB), reject a level > 8× the source voxel count, and require every input/derived float volume to fit `AL_SIZE_MAX/sizeof(float)` as well as `INT_MAX` (`INT_MAX` alone permits a wrapped float allocation on wasm32).
- **Partial-FOV overlap policy**: HEL/CR statistics use only fixed-foreground samples that map inside the moving FOV. Treating an acquisition boundary as moving background biases a cropped scan toward scale/shear transforms that pull unavailable anatomy into the grid (the FLIRT Fig-1 problem). A 10%/16-sample overlap floor still rejects degenerate fits. LS retains its historical background fill. Keep the fixed 64-chunk reduction and skip out-of-FOV samples inside each chunk so `-p1==-pN` remains bit-identical.
- **HEL joint-foreground (base-gated moving-dark skip, HEL only).** The fixed side is foreground-thresholded in `cf_make_samples`; when the FIXED/base image is itself hard-zeroed (`>25%` of its full-resolution voxels sit exactly at its minimum), the HEL cost also skips a fixed sample whose mapped MOVING intensity is at background/air (`<= m_bg + CF_HEL_MDARK_FRAC*range`, with `CF_HEL_MDARK_FRAC=0.01`). This is intentionally restricted to masked/skull-stripped bases such as `MNI152_2009_SSW`: their fixed foreground is compact real tissue, so dropping moving-air partners is surgical and removes the moving-bin-0 independence floor that otherwise flattens the Hellinger well (observed SSW pitch +33°→+12°, masked HEL 0.086→0.188). **Never apply the skip unconditionally.** With a whole-head base, broad legitimate fixed foreground can map to in-FOV moving air because the subject has less head/neck coverage; dropping those pairs makes the pose choose its own histogram denominator. The raw `nin` overlap floor does NOT prevent this because it counts in-FOV coordinates before the moving-dark drop. On the hard-zeroed `M2017` moving image (head only 21% of a 160×256×256 FOV) against whole-head `MNI152_T1_1mm`, the unconditional skip retained only ~52% of fixed foreground and collapsed masked Hellinger 0.1814→0.1233 plus brain NCC 0.4034→0.1047; the base gate restores the pre-drop fit byte-for-byte. **Do-not-regress invariants:** (1) classify the full-resolution BASE once from its exact minimum mass (`CF_HEL_MDARK_BASEFRAC=0.25`; observed whole-head MNI 0.061, avg152T1 0.0001, SSW 0.622); do not infer the gate from the moving image or pose. (2) Keep the moving threshold small: ≥0.02·range re-eats real low-signal cortex and can flip a fit. (3) HEL only; CR and LS retain their own policies. (4) the per-sample test stays inside the fixed 64-chunk reduction, preserving `-p1==-pN` bit identity. Validated whole-head cases are byte-identical to the pre-skip path, while hard-zeroed SSW cases remain byte-identical to the beneficial skip path.
- **Initialization selection is cheap and overlap-aware**: `fasthel`/`fastcr` score the supplied-affine and exact `-com` starts once at 8 mm, maximizing `(1 - cost) * covered_fixed_foreground`, then run the coarse orientation search and full descent for ONLY the winner. Default/`fastx` instead fit both frames under both costs and apply the SAME HEL dependence×overlap score to the resulting candidates. Do not multiply the minimized cost by overlap (that rewards low overlap). `-com` is a strict already-recentered override and `-nocmass` is a strict supplied-affine override; neither auto-selects. The initial MICCAI coverage was FLAIR 74.0% affine vs 97.6% COM and T1 59.7% vs 96.2%; after coarse refinement, overlap alone would incorrectly prefer the visually bad T1 affine branch (99.982% vs 99.954%), so selection must retain the statistical-dependence term rather than maximize overlap alone.
- **Translation guardrail is ±128 mm, not ±64 mm**: oblique/reoriented MICCAI FLAIR/T1 headers need about 77–78 mm of rigid-equivalent translation and produce COM seeds around 85 mm. A ±64 mm bound rejects the good seed before scoring, after which HEL can manufacture a low-overlap wrong rotation. The coarse optimizer remains local around its seeds, so the wider fail-closed guardrail does not expand the discrete search; `trans_large_y` regression-guards an 80 mm capture.
- **Rigid coarse, scale later**: the 8 mm level fits RIGID (scale locked at identity); freeing scale there let CR collapse into a spurious ~1.2× isotropic-shrink basin on wide-FOV cross-modal inputs. Global scale enters at 4 mm via a discrete bracket ({0.78,0.9,1.0,1.12,1.3}) scored at the rigid pose — required so a true envelope-edge scale (e.g. 0.75) is recovered (a continuous refine from identity can't reach a 25% downscale). Do NOT re-add a coarse scale scan or drop the 4 mm bracket.
- **Hard-zeroed-base multi-start (`CF_NSTRAT=3`).** Rigid coarse is safe for whole-head bases but can make the correct orientation unreachable for a skull-stripped base unless anisotropic scale is present at the same time (M2017→SSW: correct rigid cost 0.9433, wrong ~30° pitch 0.9241, correct orientation+scale 0.8937). Therefore a base with >25% exact-minimum voxels runs three complete trajectories: historical rigid HEL, a bounded scale×z-ratio HEL search with angular NMS, and CR coarse handed to HEL fine stages. The exploratory NMS must repeatedly select the globally lowest cost (enumeration order breaks ties), then greedily accept candidates separated by at least 20°; non-positive/non-finite separation is rejected, and in-place basin replacement can break both ordering and pairwise separation. Do not replace the bounded repeated scan with comparator-based `qsort`, which is disproportionately expensive in WASM. Every strategy reaches the 2 mm 7-DOF stage; only that winner receives 9/12-DOF polish. **Do not prune at 4 mm** (it disagreed with the 2 mm winner in 25/40 cases). **Do not add the whole-head fastx early-selected trajectory to this arbitration**: it won at 7 DOF but lost after polish on two stripped cases (T1w1mm NCC 0.477→0.145; MICCAI 0.099→−0.037). The gated method is byte-identical to the current fastx path on whole-head bases and `-p1==-pN` on hard-zeroed bases. Focused results reproduce the imported benchmark: ARC→SSW NCC 0.3357, T1w1mm→MNI-strip 0.4770, ARC→avg-strip 0.4972, MICCAI→avg-strip 0.0994. `test_coreg_fast.py` generates MNI-strip from tracked data and guards both T1w1mm and defaced/hard-zeroed ARC at NCC≥0.40, plus T1w1mm p1/p4 identity; `benchmark/hardzero.py` runs the full descriptive matrix without storing duplicate stripped templates, asserts every base actually crosses the same >25% exact-minimum gate, computes NCC only for T1 cases, and exits nonzero when that same-modal score is degenerate.
- **Audit-hardened invariants** (regression-guarded): (1) failure is atomic — `*result` written only after all levels + optimizer-ok + valid cost + affine validity; a degenerate fit returns nonzero and writes no `-savemat`. A sample rebuild first clears the prior arrays/counts; any degeneracy or partial allocation failure leaves an empty consistent state, aborts ALL remaining strategies, skips winner polish, and is fault-injection tested on a later strategy's first level. (2) rejects options it can't honor (`-cost/-warp/-interp/-source_automask/-dark_automask`); `-final` is allowed and initialization overrides are honored (`-com` arrives already recentered with auto-selection disabled; `-nocmass` disables it in the supplied frame). (3) `-savemat` serializes the REAL resolved config, never hardcoded defaults. (4) matrix-only (`-savemat`, no output) skips the reslice. (5) `cf_dims_ok` validates single-volume 3D and the platform's float-buffer byte limit on entry. (6) `cf_extract_float` scaling matches `nii_to_float`. (7) `max_dof` honored at every stage. (8) direct NEWUOA dimensions/sampling factors are checked with `size_t` before quadratic workspace arithmetic; oversize calls fail before allocation/callback.
- **`-weight <img>` — AFNI 3dAllineate-style graded weight (BOTH engines).** Base(stationary)-space weight; must share the base grid (dims + world frame, 8-corner check). Normalized to `[0,1]` (÷ max) and used **graded per voxel**, matching `3dAllineate -weight` (3dAllineate.c:4296-4303: convert-to-0..1, never binarized). Ordinary engine: `al_register`'s `weight_fname` param (threaded from `al_estimate`; the sym/mirror/sagseed/deface `al_register` calls pass NULL) → `al_load_user_weight` REPLACES `al_autoweight`, and `box_weight` is forced off so a USER weight stays graded even for box costs (only the manufactured autoweight is binarized). **Validated: `-cost hel -weight` reproduces `3dAllineate -weight` at Pearson ~0.984** on the MNI-2009 `weight/` samples, alignment-to-base matching AFNI. Fast engine (`coreg_fast.c`): the old `[0.5,1]` `CF_WEIGHT_FLOOR` soft-focus was REMOVED (unreleased); the weight is normalized `[0,1]` and applied per fixed sample. **The graded weight is applied ONLY at the finest pyramid level (lv==2, 2 mm)** — the 8 mm rigid coarse (lv==0) and the 4 mm global-scale bracket + refine (lv==1) stay UNWEIGHTED (`coreg_fast.c` `ctx.Lw = have_weight && lv>=2 ? ... : NULL`). This is the fix for the cross-modal scale-collapse: with the weight active at lv>=1 the 4 mm scale bracket, scored on the brain-concentrated ROI, selected the scale that shrinks the moving INTO the ROI (the FLIRT Fig-1 basin — observed on T2w and T1w_MICCAI2017 → the `@SSwarper` `MNI152_2009_SSW` weight, a ~1.13× isotropic shrink, det 1.45, masked HEL 0.095). The weight is meant to steer the FINE stage only; letting it re-select global scale (a DOF the unweighted coarse pyramid already resolved correctly) corrupts the fit. Deferring it to lv==2 recovers AFNI-parity (T2w masked HEL **0.153 vs AFNI 0.154**, det 0.69 vs AFNI 0.755) at full 12-DOF, and is neutral (<0.001) on the non-collapsing weighted rows (FLAIR/T1/fmri). This extends "rigid coarse, scale later" — the scale-later stage must ALSO see the full head. Reducing DOF (the AFNI `@SSwarper` `-warp shift_rotate_scale` approach) also prevents the collapse but underperforms in this engine: the shrink is near-isotropic, so only DOF=6 (rigid, scale locked) stops it, capping T2w at 0.113 with no scale correction — deferring the weight is strictly better. `-p1==-pN` bit-identical; unweighted fits unaffected (`have_weight==0` leaves `ctx.Lw` NULL). **Residual caveat:** the fast weighted cost is still more fragile than the ordinary/AFNI engine on hard cross-modal pairs; keep the out-of-ROI head attenuated (nonzero) and prefer `-cost hel` when a weighted fit must be maximally robust. main.c rejects `-weight` with `-applymat`/`-skullstrip`/preprocessing-only.
- **`-cost` is symmetric last-one-wins**: selecting `fast`/`fastx`/`fasthel`/`fastcr` clears `AL_CLI_COST` (as a later ordinary `-cost` clears `opts->fast`), so `-cost hel -cost fast` and the reverse both honor the FINAL selector (guarded by `test_regression.py` §7c) — do not drop either clear.
- Configured **only** through `coreg_fast_opts`; ambient `COREG_FAST_*` env vars are intentionally ignored so embedding applications and saved matrices are reproducible.

## Nonlinear `-qwarp` stage — `qwarp.c/.h`

An attributed public-domain port of AFNI's single operation `3dQwarp -blur 0 3 -source <moving> -base <stationary> -prefix <output>` (see the "Clean-room provenance (licensing)" boundary — this is an **attributed** port of AFNI's PD files, NOT clean-room, and NOT derived from the GPL AFNI files). Exposed as an **exclusive CLI mode**: the ONLY accepted form is `allineate <moving> <stationary> -qwarp <output>` — any extra option, missing operand, stdin/stdout operand, mismatched grid, 4D/multi-volume input, or unusable geometry is a hard error before the heavy work starts (`main.c` `run_qwarp`; no `al_opts` field). Public API is `int qwarp_run(const nifti_image *moving, const nifti_image *stationary, nifti_image **result)`.

**This is only the nonlinear stage.** The moving image must ALREADY be unifized, skull-stripped, affine-aligned, and sampled on the stationary grid (inputs must share equal dims + equivalent usable sform/qform, checked to a small tolerance). No allineate/affine step, no transform composition, no `-iniwarp`, no `_WARP` dataset output — those are deliberately deferred.

**Pipeline** (`qwarp_run`): float extract → data-dependent zeropad (base-derived, BIT-EXACT vs AFNI) → FWHM-3-voxel Gaussian blur of the source (base unblurred) → `mri_weightize` of the **padded** base → `qw_warpomatic` → a single AFNI-WSINC5 reslice of the **unblurred** padded source through the optimized warp (clamped to the source intensity range) → crop back to the stationary grid. Failure is atomic — no partial output — for every top-level allocation/optimizer failure (blur, weight, pad, and any patch-optimizer OOM via the `qw_ctx.fatal` flag). **Accepted v1 limitation:** the internal `void` preparation helpers (mask morphology `qw_mask_clust`/`erode`/`erodemany`, `qw_autobbox`, `qw_compute_pad`, `qw_cliplevel`/`qw_quantile`, `qw_clipate`) retain AFNI-style best-effort behavior on their OWN allocation failure (partial mask / fixed padding / unclipped fallback) — an OOM-only path that still yields a COMPLETE valid warp, never partial output, so the atomicity contract holds. Propagating a prep-status is deferred hardening.

**Engine** (`qw_warpomatic`, a faithful port of `mri_nwarp.c` IW3D_warpomatic for the default 3dQwarp config): level-0 basis escalation SINCC(3)→CUBIC_LITE(12)→CUBIC(24)→QUINTIC_LITE(30)→QUINTIC(81) over one global patch (no penalty), then a shrinking-patch schedule (`0.749999`/level, forced-odd widths, ~50% overlap, alternating sweep parity) down to the minimum patch (Hngmin=25); ordinary levels use CUBIC_LITE. The internal displacement is an `IndexWarp3D` (base index → source sampling index, voxel units). Similarity is clipped-Pearson (PEARCLP; plain-Pearson fallback if either image has negatives). The deformation penalty (hexahedron bulk+shear energy, `max(je-1,0)^4+max(se-1,0)^4`, scaled `Hpen_fff·(Σ)^0.25`) ramps on at level ≥ 3. Reuses `powell_newuoa_con`.

**Do-not-regress gotchas** (each was real):
- **Double-warp bug**: `qw_hwarp_apply`'s final source interpolation MUST sample `srcim_blur` (the ORIGINAL blurred source, = AFNI `SRCIM`) at the composed location — NOT the already-warped `aasrcim`. Sampling `aasrcim` double-applies the warp; the symptom is per-level cost INCREASING. `aasrcim` is also re-warped from scratch each level start (`IW3D_warp_floatim(Haawarp, SRCIM, LINEAR)`) to reset compose-interpolation drift.
- **Box/ball deviation** (documented, unavoidable): the niimath-owned `powell_newuoa.c` drop-in exposes no box/ball constraint setter, so `qw_ctx.con` EMULATES AFNI's box/ball selection for the per-basis max-displacement scale but the optimizer always runs box-constrained. A numerical (not byte) match — do NOT "fix" it by editing `powell_newuoa.c`.
- **Weightize the PADDED base** (not "weightize then pad"), matching AFNI. Clip params for PEARCLP come from the BLURRED source (= `SRCIM`). Cost samples the blurred-warped source; only the final reslice uses the unblurred source.
- **Equivalent, not bit-identical**: the niimath Gaussian ≠ AFNI's GPL FIR blur and the clean-room median ≠ AFNI's GPL `qmed`, so validate by Pearson/RMSE vs the AFNI `usaq*` refs (`test/qwarp_compare.py`), never by bit-equality. **`-p1==-pN` is NOT required for qwarp** (OpenMP may reorder FP as AFNI's does); serial-first, OpenMP on the three hot loops (`qw_hwarp_apply`/`qw_incor_addto`/`qw_load_energy`).
- **Runtime**: ~3.5 min per 1mm case (~5 effective cores), ~24 s for the 2mm dev pair, peak RAM ~2.2 GB. Run cases ONE AT A TIME — two concurrent runs each spawn nthreads=ncores and oversubscribe OpenMP (≈4× slowdown).
- **Count/byte guard**: `qwarp_run` rejects a padded volume above either `INT_MAX` voxels or `AL_SIZE_MAX/sizeof(float)` — several inner routines take `int` counts, while the latter bound prevents wrapped float allocations on 32-bit targets. Real brain grids are nowhere near either limit; the guard mirrors the fast engine's policy.
- **Perf is memory-latency-bound (do-not-re-chase)**: the hot loops (`qw_load_energy`, `qw_incor_addto`, `qw_hwarp_apply`) are dominated by scattered gathers over the float displacement/image fields. Compute micro-opts (fast/inverse cbrt, reciprocal-lookup for the clip divide, `-mcpu=native`) all measured ~0% and were reverted; they also perturb the chaotic optimizer. Real gains would need data-layout/cache work (AoS xyz displacements, cache-blocking) — high-risk, not low-hanging.
- Env `QW_VERB=1` prints a per-level trace (zero release overhead); `-DQWARP_CHECKPOINT` enables `/tmp/qwchk_*.nii` intermediate dumps + corr prints (dev-only).

**Validation & gates.** Validated vs AFNI `3dQwarp` references (MNI-1mm) at Pearson 0.98+ (chris 0.993 / ds000031 0.996 / MICCAI 0.983), alignment-to-base matching AFNI within ~0.001. `qwarp.c/.h` are now in niimath behind the `QWARP=1`/`ENABLE_QWARP` gate and in `make parity`'s `SHARED_STABLE` (byte-identical; niimath `-qwarp <base>` output matches allineate at Pearson ~0.998 cross-compiler). `make test` covers: the exclusive CLI grammar; a synthetic end-to-end warp-improvement check (output finite, source-range-bounded, beating the unwarped source's Pearson-to-base by a fixed margin); and `test/test_qwarp_capi.c`, a C-API boundary test asserting the atomic-failure contract (`*result==NULL` on oversize/too-small/mismatched/no-transform/both-unusable metadata), the dual-coded sform→qform fallback + higher-code preference, and NEWUOA-`mfac`/`g_qw` restoration after a successful optimize. The AFNI-comparison acceptance gate is the separate gitignored `test/qwarp_compare.py` (Pearson floor, normalized-RMSE ceiling, anti-trivial `cand↔ref > both base` relation); its 1mm reference dataset is NOT bundled (it lived in the removed gitignored `qwarp/` dev folder — regenerate with AFNI when needed). Serial-UBSan + `leaks` clean. Deferred: OOM-only prep-helper status propagation; large-volume reslice scratch / cache-oriented displacement layout.

## Anonymization `-reface` stage — `reface.c/.h`

Emulates AFNI `afni_refacer2.csh -mode_reface`: `allineate <moving:subject> <stationary:template> -reface <shell> <output>`. Registers the subject to the template (either engine), back-projects a template-space face-replacement **shell** onto the ORIGINAL subject grid, and composites an anonymized image on the subject grid. The template is the STATIONARY image (not the moving) so the template's registration weight can be supplied via `-weight` (AFNI uses `-base template[1] -weight template[2]`). **This is CLEAN-ROOM, not an AFNI port** — the arithmetic recipe is trivial and the blur is niimath's own `nifti_smooth_gauss_f32` used as normalized convolution, NOT AFNI's `3dBlurInMask` (see "Clean-room provenance").

**Shell value semantics** (a template-space int/float image, any FOV sharing the template world frame): `>0` = insert (scaled), `==0` = preserve subject, `<0` = zero. The AFNI 4D shell `afni_refacer_shell_sym_2.1` has subbrick `[0]`=reface_plus (face+ears+skull) and `[1]`=reface/face; the CLI takes a 3D file, so the user picks the subbrick by naming it (extract `[1]` — the mode_reface/face shell — for the standard reface; `[0]` for reface_plus).

**Pipeline** (`main.c do_reface` → `reface.c reface_apply`): capture `subject_orig` BEFORE any `-robustfov` crop → register (fast default / ordinary `-cost`) → `nii_apply_affine(shell, subject_orig, inverse(fixed_to_moving), NN, 0)` back-projects the shell onto the subject grid (the world-mm matrix reuses the existing apply path; NN preserves the −1/0/+ labels) → `reface_isola` drops isolated speckle → `reface_apply` computes `ifac`, the sign-based composite, and the masked blur. Output float32 on the exact subject grid.

- **`ifac` brightness match** (AFNI `3dBrickStat -non-zero -mean`): `ibar = mean(subject over {shell>0 && subject≠0})`, `mbar = mean(shell over {shell>0})`, `out = shell>0 ? shell·ifac : shell<0 ? 0 : max(subject,0)`. Empty overlap / non-finite / `mbar≤0` FAIL closed (no misscaled face). Fixed-order scalar accumulation → deterministic.
- **Masked edge blend**: normalized Gaussian (FWHM 2.666 mm) inside the `shell>0` mask — `blur(out·mask)/blur(mask)` where mask, else `out` (AFNI `3dBlurInMask -preserve` equivalent). Two `nifti_smooth_gauss_f32` calls; the separable blur parallelizes only across independent rows, so `-p1==-pN` is bit-identical (the whole reface op is).

**Do-not-regress invariants** (the key ones regression-guarded in `test_regression.py` §14/§14b):
- **Output is the ORIGINAL subject grid + RAW intensities.** `subject_orig` is captured before `-robustfov`; the delivered image is never cropped. `-robustfov` affects only the fast-engine ESTIMATE (its returned world matrix is valid for the raw image; `-robustfov` preserves retained-voxel world coordinates, so a cropped-grid fit back-projects correctly onto the uncropped subject).
- **Shell interp is fixed NN, fill 0.** `-final`/`-interp`/`-fill` are REJECTED (not silently ignored). Genuine out-of-shell-FOV subject voxels get 0 → preserved by the `==0` branch.
- **`ifac` uses the PROVIDED 3D shell's positive region**, not AFNI's always-subbrick-`[0]` region (AFNI computes `ifac` from `[0]` even for mode_reface `[1]`; the 3D-file API computes it from whatever shell is passed — an accepted, documented divergence).
- **Fast-only option** `-robustfov` is rejected with an explicit ordinary `-cost`, and default-fast fallback to Hellinger is suppressed when it is supplied (it is not honored on the ordinary path — fail rather than silently change semantics).
- **Tight option whitelist.** Accepts ONLY `-cost`, `-weight`, `-p` + the fast-only trio. The ordinary registration-tuning knobs `-warp`/`-cmass`/`-nocmass`/`-source_automask`/`-dark_automask`/`-zoom`/`-nosagseed` are REJECTED (one consolidated check; `-nosagseed` via `opts.sagseed==0`) — several of them set `fast_incompatible` and would otherwise silently switch reface onto the ordinary engine, or silently alter the fit.
- **Public-API overflow guard.** `reface.c` `reface_nvox()` is an overflow-checked voxel count used by BOTH `reface_apply` and `reface_isola`; it rejects dims that wrap `size_t` (incl. the `*sizeof(float)` mallocs) AND caps at `INT_MAX` (the Gaussian backend takes `int` counts) so an oversize C-API image fails cheaply before any scan/alloc. `do_reface` requires `al_image_xform(shell)` SUCCESS (a usable coded form), not merely `sform_code>0` — a coded-but-singular shell is rejected rather than placed via the pixdim fallback.
- **Rejects** `-skullstrip`/`-applymat`/`-master`/`-savemat`/`-com`/`-sym[/d/b]` (fixed output semantics; header seeds would make the fitted matrix relative to a modified subject header), stdin shell, and missing stationary/output. **`-sagseed` is intentionally NOT in the reject list**: it defaults to 1 (`al_opts_default`), so adding `opts.sagseed` to the condition would reject EVERY reface run; and `-sagseed` only acts inside the (rejected) `-sym` block, so it can never modify the subject header on the reface path. Do not "fix" this by adding it.
- **Ownership** (`main.c`): reface mode retains `subject_orig` (freed with `moving`); fast leaves `moving` unmutated (composite the pristine subject). The ordinary path uses `nii_allineate_estimate` (NOT `nii_allineate`) — it fits WITHOUT the reslice-then-discard the full call would do and does not mutate `moving`. `reface_apply` mutates neither input and returns a NULL result on any failure (atomic). `subject_orig` is freed on every reface exit path (audited) and is NULL on non-reface paths so the frees are safe.
- **Face-coverage gate (anonymization safety, FAIL-CLOSED).** The design trusts the registration (as AFNI does), so a "successful" but grossly-wrong fit could land almost none of the shell's face region on the subject and write an output whose face is NOT removed. `do_reface` cannot verify the fit but measures how much of the face PHYSICAL VOLUME mapped into the subject FOV: the template and subject voxel volumes are each `|det|` of the image's own coded world transform (`reface_voxvol_mm3`, `xyz_units`-normalized to mm³) — NOT the pixdim product, which is wrong for a scaled/sheared sform and which the previous version accidentally cancelled to a raw cross-grid count ratio. **Below 10% coverage it FAILS CLOSED (refuses to write), matching niimath.** This is a deliberate privacy posture: for an anonymization command a possibly-unanonymized output is worse than a hard error, so we accept that a legitimate but very-partial-FOV subject may be rejected and need re-running with a better fit/template. (Zero overlap also fails in `reface_apply` via `mcnt==0||icnt==0`.) Real whole-head data reports ~83–85% coverage. **Keep this gate byte-aligned with niimath's `nifti_reface_wrap` (`reface_voxvol_mm3`, fail-closed at <10%);** the two drivers are non-shared but their coverage semantics must match. **Anonymization output must still be QC'd** (AFNI generates QC montages; this port does not).
- **Finiteness under `-ffast-math`.** `reface.c` uses magnitude guards (`reface_finitef`/`reface_finited`: `fabs(v) <= FLT_MAX/DBL_MAX`), NEVER `isfinite()` — the mandatory `-ffast-math -fno-finite-math-only` can fold `isfinite()` to true. A NaN subject voxel (`NaN != 0` passes the `ifac` accumulation filter) is caught by the double guard on `ibar`/`mbar`/`ifac` and by the final float output guard.

**Validation** (measured vs the AFNI `afni_refacer2` reference on a whole-head T1; the dev dataset lived in the removed gitignored `reface/` folder and is not bundled): fast weighted — the canonical `-weight <SSW_weight>` command — Pearson 0.988 / nRMSE 0.021; plain fast 0.966; ordinary `-cost lpa` 0.977; output geometry byte-matches the subject. **Ordinary 0.977 vs a predeclared 0.98 floor is an ACCEPTED deviation, not a bug:** the gap is AFNI's registration-specific tuning (`-cmass -source_automask+4 -fineblur 3 -twobest 3 -num_rtb 33 -warp shift_rotate_scale` + `3dUnifize -GM`) that the "engine-defaults" decision deliberately does not replicate; the canonical fast+weight command (0.988) clears its floor. `make test` §14 asserts region semantics/geometry/`ifac`/isola/`p1==pN`/the rejection matrix + ordinary(`ls`)/weighted success on a synthetic identity fixture (`-cost fastcr` — HEL is imprecise on smooth phantoms); §14b adds a NON-IDENTITY (known-translation) fixture that catches a reversed `inverse(fixed_to_moving)` an identity fixture cannot (it verifies the shell's zero-out block removes the SUBJECT-side brain, not the template side). Serial-UBSan clean; `leaks` clean on the fast release path (the ordinary engine's ~227 MB process-global retention is pre-existing and reface-independent). Deferred: AFNI datatype re-shortization (always float32); `deface`/`reface_plus`/`cubeface` modes; header-seed/`-savemat` composition.

## Architecture (file map)

- **allineate.c** (~5300 lines) — the allineate engine: BLOK/TOHD local correlation, costs (Hellinger default / Pearson / lpc / lpa), coarse→fine twopass NEWUOA, affine warp + interp, OpenMP thread-local workspaces. The coarse driver is an AFNI-derived port — notably the matching-point floor `AL_NPT_MATCH_MIN` MUST stay at AFNI's **98765** (fewer coarse samples make the Hellinger surface too noisy and a cross-modal case lands in a wrong basin from the header start). **Hardening — histogram-OOM must not score as a win:** `build_2Dhist` is `void`; a malloc failure leaves an empty histogram that `al_helmicra` maps to cost `0.0` (below `AL_BIGVAL`), which the fail-closed abort would accept as success under sustained OOM. A thread-local `al_hist_oom` flag (set ONLY at the two malloc-fail returns, cleared at entry, consumed in `GA_scalar_costfun` → `AL_BIGVAL`) routes OOM to the abort. Do NOT set it on the legitimate empty-histogram returns (`n<=9`/constant/`ngood==0`) — those must keep returning `0.0` or the default trajectory (and p1==pN bit-identity) changes.
- **allineate.h** — public API + `al_opts` + inline enum parsers. Engine selector uses named `AL_ENGINE_*` constants (0=allineate, 1=fast/CR, 2=fast/HEL).
- **nifti_io.c/.h** — byte-for-byte niimath NIfTI-1/2/Analyze I/O, gzip/zstd, and mat44/quaternion.
- **powell_newuoa.c** — NEWUOA derivative-free optimizer (f2c).
- **miniCoreFLT.c/.h** — demo-only `nifti_robustfov`/`nii_ensure_float32` plus the float32 typed Gaussian routines from niimath `coreFLT.c`, exported as `nifti_smooth_gauss_f32` through the byte-identical `core32.h`. Add further coreFLT integration here, not in shared registration files.
- **coreg_fast.c/.h** — the fast engine (above); hard-zeroed-base robustness combines the base-gated HEL joint-foreground moving-dark skip with the gated three-strategy coarse multi-start.
- **qwarp.c/.h** — the `-qwarp` nonlinear stage (below): an attributed public-domain port of AFNI's `3dQwarp -blur 0 3` (see the qwarp section under "Key design gotchas" and the "Clean-room provenance" licensing boundary).
- **reface.c/.h** — the `-reface` anonymization compositing stage (below): a CLEAN-ROOM reimplementation of the AFNI `afni_refacer2.csh -mode_reface` recipe (`reface_isola` + `reface_apply`). NOT an AFNI port — the arithmetic recipe is trivial and the edge blend uses niimath's own `nifti_smooth_gauss_f32` (via `core32.h`) as a normalized convolution, not AFNI's `3dBlurInMask`. Now back-ported to niimath (`-reface <tmpl> <shell> <weight>`, weight REQUIRED, via `coreFLT.c` `nifti_reface_wrap`) and in `make parity`'s `SHARED_STABLE` (byte-identical; niimath reface output byte-matches allineate on the synthetic fixture).
- **main.c** — CLI `<moving> [stationary] [opts] <output>` (stationary optional → preprocessing-only); opts parse on either side of the output. Hosts `-reface` (`do_reface`, a `main.c`-local `reface_name`).

## Key design gotchas

- **Ordinary-engine far-origin capture follows AFNI without auto-COM.** AFNI derives translation bounds from a zero-padded fixed FOV; the old compact bound stopped near 70 mm while the MICCAI scans need about 82–86 mm. `al_register` computes AFNI's padded dimensions and uses the expanded range only when the brightness-centroid displacement is within 10% of an unpadded boundary. The centroid is only a range diagnostic: it never changes the initial pose, so `-cmass` still explicitly requests a COM start and `-nocmass`/default still use the supplied affine. Do not hoist the fast engine's `(1-cost)×overlap` seed selector into this path; both ordinary starts were trapped by the old range, and expanding every ordinary case perturbs the compact port's normalized random search. The 78 mm shift regression guards this behavior.
- **AFNI-sized coarse candidate pool matters on large moving FOVs.** `PARAM_MAXTRIAL` is 29, and when moving physical volume exceeds 1.3× fixed volume the ordinary engine retains 29 candidates instead of five, matching AFNI's volume-triggered `twobest` policy. Even normal cases use the corresponding 88-entry coarse pool. Lowering this restored speed but degraded both the revoxelized affine fixture and MICCAI T1 accuracy.
- **Cross-modal weight/clip semantics (the T2w→T1 shrink fix).** On a whole-head cross-modal source the allineate engine used to shrink the moving brain (det>1, foreground pushed out of the source FOV) where AFNI contracts to fill — the FLIRT Fig-1 overlap degeneracy. The current design, matching AFNI:
  - **Autoweight is BINARY for box-mode costs** (Hellinger/MI/NMI/CR), like AFNI `-autobox`: binarize `wght` (→1.0 in-mask) so every in-mask voxel weighs equally. Only `ls/lpc/lpa` use the graded intensity weight. A graded weight down-weights the cortical periphery, so pushing it out of FOV costs little and the overlap shrinks unpenalized — this was the dominant divergence. Do NOT make the weight graded for box costs.
  - **Histogram membership uses `mri_topclip`**, stored separately from the clipate edge-bin clips (`aj_topclip`/`bs_topclip`, threaded through `al_helmicra`→`build_2Dhist`). Membership DROPS pairs above each image's topclip (bright non-brain a T1 template lacks); edge-binning still uses the clipate clips. Do NOT use the lower clipate `ctop` for membership (over-drops, breaks same-modality).
  - **Edge-bin clips are sample-based** (AFNI `set_2Dhist_xyclip`): a per-stage `need_hist_setup` flag, consumed by the stage's FIRST **sequential** cost eval which caches the clip; parallel evals read the cache. EVERY stage that arms the flag needs a sequential warm-up eval before its parallel region, or the shared write races and breaks p1==pN bit-identity.
  - These are gated on a `hist_cost` bool so `ls`/`lpc`/`lpa` skip the extra quantile/warm-up; the default Hellinger path is byte-for-byte unchanged. `AL_VERB` prints per-stage clips + pose (0 release overhead). **Validate any change here on T2w AND T1 AND fmri AND `-skullstrip` + p1==pN bit-identity + `make test`.**
- **Image transforms**: `al_image_xform()` prefers sform when `sform_code >= qform_code`, otherwise qform, and falls back from an unusable preferred form to the other coded form. The shared `al_image_xform_or_pixdim()` applies the single no-form policy used by registration, reslicing, `-sym`, `-sagseed`, and `-com`: if neither coded form is usable, synthesize a pixdim-centered frame.
- **Serial-only**: process-globals (`gstup`, `g_last_*`, `g_cf`, ...). Never call registration concurrently across host threads.
- **Source is modified in-place** during registration (data replaced, dims → base grid).
- **`-com` / centroid**: filters non-finite voxels with `al_finitef` (NOT `isfinite` — under fast-math one NaN would poison the accumulator → NaN header). Positivity brightness filter + geometric-center fallback.
- **qform-sync**: every read fills an absent/degenerate sform from a valid qform; never overwrites a valid sform.
- **`-interp` vs `-final`**: `-interp` = fine-pass matching interp (default linear); `-final` = output warp interp (default cubic; linear for `-skullstrip`). Resolved once via `resolve_output_interp()`.
- **`-fill auto|zero|nan`** (`al_opts.fillmode`, `AL_FILL_*`): out-of-FOV fill for the output reslice, ported **byte-identically from niimath** (`al_resolve_fillv`/`al_image_fillv` and the fused-warp `al_fill_fused_oob` path). `auto` (default) = 0 unless the float32 source minimum is negative (CT/HU air), then that minimum; `zero` = always 0; `nan` = NaN. The ordinary engine honors `al_opts.fillmode` internally; the fast, `-applymat`, and `-master` reslices route through `resolve_output_fill()` in `main.c` (which calls `al_image_fillv` so AUTO detects negative air even in a raw int16 CT). The AUTO default keeps positive-only (MRI) output byte-identical to the historical hardcoded-0 fill.
- **`-robustfov`** (FSL clean-room port): crops top-of-head-down; shifts a usable coded sform/qform. **Pair with `-cmass` on the allineate engine** — cropping removes the inferior anchor the header-started search needs, so without a COM init the fit drifts into an inflated-scale basin. This is a search-INIT effect, NOT a geometry bug — do NOT "fix" the crop math or re-add auto-cmass. The fast engine is immune (always scores a COM seed).
- **`-skullstrip`** (`nii_deface`): registers stationary→template (the well-posed direction), inverts, warps the template-space mask onto the stationary grid, zeros non-brain to the image minimum.
- **Plain-float32 conversion** (`al_ensure_float32`, local to `allineate.c`): a float32 image with a pending `scl_slope`/`scl_inter` is still converted via `nii_to_float` so mask fill reads physical intensities (a negative slope can't then paint masked voxels bright).
- **`-sym`/`-symd`/`-symb`** (MSP alignment via a world-X mirror fit + half-transform): `-symd` snaps the frame to the nearest orientation-preserving signed permutation (handedness folded into the score). `-symb` prefers the world frame on ties (de-oblique must beat it by > 0.5°). Compares correction rotation only, not fit cost (accepted tail-case limitation).
- **`-sagseed`**: 3-DOF constrained fit (y-shift/z-shift/pitch); on by default when `-sym` runs with a template.
- **`-savemat` / `-applymat`**: world-mm `fixed_to_moving`; `-applymat` reslices onto any target sharing the fixed frame. Caveat: if `-com/-sym/-sagseed/-zoom` seeded the moving header, the matrix is relative to that seeded header (so `-applymat` reproduces a plain fit exactly, not a pre-stepped one).
- **`-master grid`**: register at the stationary resolution, reslice the result onto `grid` (must share the stationary world frame) instead of the stationary grid. The fit is unchanged (same matrix); output is byte-identical to `-savemat`+`-applymat`-onto-`grid`. The fast engine reslices its unmutated `moving` straight onto the grid; the allineate engine mutates `moving` in place, so main.c **deep-copies** (`nim_deep_copy`) the moving image after any pre-steps and re-reslices the copy. Rejected with `-applymat`/`-skullstrip`.
- **`-dark_automask`**: drops matched pairs at either image's darkest value (per-image min, safe for signed CT). Off by default.
- **`-zoom`** (infant/abnormal size): widens the seed + main-fit scale range to [0.5, 2.0]; ordinary fits stay byte-identical.
- **Piping** (`-`, demo-only host behavior over shared niimath I/O): only the moving image may be stdin; output may be stdout as uncompressed NIfTI-1.

## Clean-room provenance (licensing)

The fast path is designed only from published descriptions (SPM `spm_coreg`; FLIRT, Jenkinson & Smith 2001 PMID 11516708; FreeSurfer, Reuter et al. 2010 PMID 20637289) — **never from `niimath/src/GPL/`** (hard rule). The Gaussian backend in `miniCoreFLT.c` is the BSD/public-domain float32 typed routine from niimath `coreFLT.c`; `coreg_fast.c` calls it through the shared `core32.h` declaration.

**Two DIFFERENT provenance regimes coexist — do not conflate them:**
- **Clean-room (the fast engine, `coreg_fast.c`)**: designed from published papers only, never from AFNI/GPL source. This is a HARD boundary because AFNI's registration code is GPL.
- **Attributed public-domain port (`qwarp.c`, the `-qwarp` nonlinear stage)**: a DIRECT port of AFNI's **public-domain** files (Robert W. Cox / "Zhark", NIMH/NIH) — `3dQwarp.c`, `mri_nwarp.c`, `thd_incorrelate.c`, `thd_cliplevel.c`, `thd_automask.c`, `edt_volpad.c`, `mri_genalign_util.c` — used WITH attribution, pinned at AFNI rev `506e48403` (`AFNI_26.1.00-6`). AFNI's `LICENSE.txt` is a NIH Public Domain Notice; direct porting of those files is permitted. **BUT AFNI has GPL-v2 / Medical College of Wisconsin EXCEPTIONS** — the following AFNI files/functions are GPL, NOT public domain, and are **explicitly EXCLUDED from the port** (never ported, never copied):
  - `edt_blur.c` → `FIR_blur_volume_3d`, `mri_float_blur3D`, `fir_gaussian_load`, `fir_blur{x,y,z}` (FIR Gaussian blur).
  - `edt_buildmask.c` → `MCW_build_mask` (spherical neighborhood offset list).
  - `cs_qmed.c` → `qmed_float` and the `median_floatN` helpers (quickselect median engine).

  (`mri_medianfilter.c`'s driver is public domain, but it CALLS the two GPL functions above, so the median step is reimplemented, not ported.) Their function is instead supplied by **niimath's own BSD/public-domain `coreFLT.c`** — the Gaussian blur via `nifti_smooth_gauss_f32` (already in `core32.h`) — and by small clean-room routines in `qwarp.c` (the sphere-offset list and the median selection are standard algorithms written independently). Verify each AFNI source file's own header before porting from it (a "Medical College of Wisconsin" copyright ⇒ GPL ⇒ substitute; no such header ⇒ NIH public-domain core ⇒ port with attribution). Never copy from a GPL AFNI file or from `niimath/src/GPL/`. The optimizer `powell_newuoa.c` is the same public-domain Powell NEWUOA AFNI uses.
